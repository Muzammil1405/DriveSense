> DriveSense uses computer vision and facial landmark analysis to detect driver drowsiness and distraction in real time — no specialised hardware required. Just a standard webcam.

---

## 📌 Problem Statement

Driver fatigue and inattention account for over **20% of road accidents** worldwide. Existing solutions are expensive or require dedicated hardware. DriveSense provides an affordable, software-only solution using a standard laptop webcam.

---

## 🎯 Features

| Feature | Method | Threshold |
|---|---|---|
| Drowsiness detection | Eye Aspect Ratio (EAR) | EAR < 0.25 for 20+ frames |
| PERCLOS monitoring | Rolling eye-closure % | > 35% closure in 100 frames |
| Blink rate tracking | Rapid EAR dip detection | Shown in HUD |
| Distraction detection | 3D Head Pose Estimation | Yaw > 25° or Pitch > 20° |
| Two-level alerts | Warning → Danger | Sound + colour overlay |
| Real-time HUD | OpenCV overlay | FPS, EAR, PERCLOS, angles |

---

## 🧠 How It Works

### Eye Aspect Ratio (EAR)
Based on the paper by *Soukupová & Čech (2016)*. For each eye, EAR is calculated from 6 facial landmarks:

```
EAR = (‖p2–p6‖ + ‖p3–p5‖) / (2 × ‖p1–p4‖)
```

- **Open eye** → EAR ≈ 0.30–0.40
- **Closed eye** → EAR < 0.25

If EAR stays below threshold for 20+ consecutive frames, a drowsiness alert fires.

### Head Pose Estimation
Using `cv2.solvePnP()` with 6 stable facial landmarks, the system estimates:
- **Yaw** — left/right rotation
- **Pitch** — up/down tilt
- **Roll** — head tilt

If yaw > 25° or pitch > 20° for 20+ frames, the driver is marked as distracted.

### PERCLOS
A rolling window of 100 frames tracks what percentage of the time the driver's eyes are closed. >35% triggers a high-severity alert.

## ⚙️ Installation

### Prerequisites
- Python 3.9 or higher
- A webcam

### Step 1 — Clone the repository
```bash
git clone https://github.com/YOUR_USERNAME/DriveSense.git
cd DriveSense
```

### Step 2 — Create a virtual environment
```bash
python -m venv venv

# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate
```

### Step 3 — Install dependencies
```bash
pip install -r requirements.txt
```

### Step 4 — Generate the alert sound (run once)
```bash
python generate_alert_sound.py
```

## ▶️ Usage

```bash
# Default webcam
python main.py

# Second camera
python main.py --cam 1

# Run on a video file
python main.py --video path/to/driver_video.mp4

# Disable sound
python main.py --no-sound
```

## ⚡ Configuration

All thresholds are in `utils/config.py`. Adjust to your webcam and environment:

```python
EAR_THRESHOLD     = 0.25   # Lower = stricter eye-closed detection
EAR_CONSEC_FRAMES = 20     # Increase for fewer false drowsiness alerts
YAW_THRESHOLD     = 25     # Degrees allowed before "distracted"
PERCLOS_THRESHOLD = 0.35   # 35% eye closure in rolling window
```

## 🔬 Research Basis

- **EAR Method**: Soukupová, T. & Čech, J. (2016). *Real-Time Eye Blink Detection using Facial Landmarks.* CVWW.
- **PERCLOS**: Wierwille, W.W. & Ellsworth, L.A. (1994). *Evaluation of driver drowsiness by trained raters.*
- **Head Pose via PnP**: Lepetit, V., Moreno-Noguer, F., & Fua, P. (2009). *EPnP: An Accurate O(n) Solution to the PnP Problem.*
